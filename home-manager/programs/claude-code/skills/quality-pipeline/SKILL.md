# 品質パイプライン

## 実行順

1. **テスト**: 全テストを実行し、緑であることを確認
2. **Lint**: 静的解析を実行。警告・エラーはゼロにする
3. **Format**: 自動フォーマッタを適用し、差分があれば保存
4. **再テスト**: Format により意図せず壊れることがないかを検証

## 失敗時の扱い

- **テスト失敗**: 直ちに TDD サイクルの Red フェーズに戻る。失敗テストを特定し、原因を修正してから再実行
- **Lint 警告**: 修正できるものは修正、意図的な例外は理由コメントで明示 (Code comments explain "why not")
- **Format 差分**: そのまま適用。手動整形で無駄に戦わない

## 禁止事項

- テストをスキップ・無効化して通す (例: `xit`, `t.Skip`, `@pytest.mark.skip`) — 理由を明記した一時的なもののみ許容し、次サイクルで解消
- Lint ルールの広域無効化 (`// eslint-disable-next-line` のような局所的な例外のみ許容)
- Format の部分的無効化
- pre-commit / CI フック回避 (`--no-verify`)

## 完了判定

全仕様でテストが緑、Lint クリーン、Format 適用済みの状態を "実装完了" とみなす。
